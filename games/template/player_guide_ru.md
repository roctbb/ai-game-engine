# Шаблон игры
## Общий формат карты

Во всех задачах с полем карта передается как список столбцов: первая координата - `x`, вторая - `y`.

```python
width = len(board)
height = len(board[0]) if width else 0
cell = board[x][y]
```

`x` растет вправо, `y` растет вниз. Перед проверкой клетки всегда проверяйте границы: `0 <= x < width` и `0 <= y < height`.

Это технический пример turn-based игры для разработки новых задач и лобби.

Напишите функцию `make_choice(state)` или `make_move(state)`, которая возвращает одно из действий: `inc`, `dec`, `stay`, `stop`.

В `state` приходят:

- `turn`: номер хода.
- `value`: текущее значение счетчика.

Цель демо-игры: управлять счетчиком и при необходимости остановить симуляцию действием `stop`.

Для отладки можно использовать `print(...)`. Вывод появится в консоли под визуализацией синхронно с кадрами replay.

## Подробный разбор

### Что нужно написать

Это соревновательная игра: несколько участников запускаются на одной карте, а результат сравнивается по очкам, победителю или ничьей.

Главный контракт: `Используйте этот пакет как основу для новой turn-based игры.`

Перед тем как писать сильную стратегию, убедитесь, что функция всегда возвращает действие правильного типа. Даже слабый, но стабильный бот лучше, чем бот, который иногда падает с ошибкой.

### Как читать карту и координаты

Это шаблон для новых игр. В нем важнее всего внимательно описать входные данные будущей игры и не смешивать разные роли в одной функции.

### Первый рабочий алгоритм

1. Сохраните все разрешенные действия в список.
2. Для каждого действия посчитайте новую позицию.
3. Отбросьте действия, которые ведут в стену, ловушку, воду, огонь, тело змейки или другую запрещенную клетку.
4. Из оставшихся выберите действие, которое приближает к ближайшей полезной цели.
5. Если полезной цели не видно, выберите любой безопасный ход, чтобы программа продолжила игру.

Для этой игры хорошая основная идея: жадная стратегия с пересчетом цели каждый ход: выбирайте полезную цель и проверяйте, не опередит ли вас соперник.

### Как сделать решение сильнее

В матче участники часто ходят одновременно. Поэтому полезно думать в два шага: что я получу этим ходом и что сможет сделать соперник после моего хода. Если в результате есть `scores`, `winner_slots`, `placements` или `is_tie`, используйте их при сравнении стратегий.

Полезный прием для отладки: печатайте короткие сообщения, например выбранную цель, оценку вариантов и итоговое действие. Тогда по replay легче понять, почему бот сделал именно этот ход.

### На что обратить внимание

В соревновании хороший ход должен быть не только выгодным вам, но и неудобным для соперника.

Для первого самостоятельного улучшения попробуйте сравнить свою стратегию с демо-ботом и посмотреть, в каких ситуациях она теряет очки. После каждого изменения запускайте пример несколько раз: карты генерируются случайно, и стратегия не должна зависеть от одного заранее угаданного маршрута.

### Где смотреть примеры

## Быстрый старт и простой пример

Эта игра - соревновательная игра. Начните с простого решения, убедитесь, что оно запускается, а потом постепенно улучшайте стратегию.

### Контракт функции

Используйте этот пакет как основу для новой turn-based игры.
Функция make_choice(state) может вернуть одно из действий: inc/dec/stay/stop.

Координаты в играх с сеткой читаются одинаково: `x` растет вправо, `y` растет вниз. Если поле называется `board` или `maze`, карта хранится как список столбцов: `len(board)` — ширина, `len(board[0])` — высота, клетку читают как `board[x][y]` или `maze[x][y]`. Если в игре используется `state`, внутри него обычно есть текущая позиция, карта и цель.

### Минимальный пример

Этот пример специально простой. Он показывает формат функции и ответа, но не пытается быть сильной стратегией.

```python
def make_choice(state):
    if state["turn"] < 5:
        return "inc"
    return "stop"
```

### Как улучшать решение

1. Сначала выпишите, какие данные реально приходят в функцию: координаты, карта, ресурс, счет, роль или `state`.
2. Сделайте проверку допустимых ответов: бот не должен ходить в стену, открывать уже открытую клетку или строить башню без бюджета.
3. Для задач с картой заведите маленькие функции проверки границ и чтения клетки через `board[x][y]`.
4. Если есть ключи, время, вода, огонь, батарейка, ящики или соперник, храните это в состоянии алгоритма, а не только координаты.
5. Сравните свое решение с примером из `examples`, запустите несколько случайных карт и посмотрите replay.

### Где посмотреть полный пример

- `examples/bot_counter.py` для слота `bot`: Демо: счетчик.

Полный пример можно открыть в папке `examples` рядом с этой документацией. Его удобно запускать как рабочую основу, а затем упрощать или усиливать под свою стратегию.


## Баланс матча

В соревновательных играх ходы участников считаются одновременно. Если несколько игроков набрали одинаковый счет, в результате может быть ничья: смотрите `winner_slots`, `is_tie`, `scores` и `placements`.


## Ошибки и подсказки

Если бот возвращает неправильную команду, идет в стену или ему не хватает ресурса, движок добавляет событие с полем `message`. Это короткое объяснение можно показывать участнику в интерфейсе.
