# Гравитационные яблоки

## Раздел

**Симуляции мира**. Здесь не нужен BFS, графы или динамическое программирование. Главная цель - пошагово обновлять матрицу, пока мир не станет устойчивым.

## Цель игры

Яблоки падают вниз в своих столбцах, пока под ними пустая клетка. Камни `#` блокируют падение. Нужно вернуть устойчивое поле.

## Что видно в визуализации

Сад показан сбоку. Яблоки падают вниз по клеткам, камни блокируют падение, а визуализация показывает промежуточные состояния после отдельных падений. Так видно, почему одного прохода по карте недостаточно.

## Что написать

Определите `simulate(board)`. Верните новую матрицу после падения яблок.

## Входные данные

- `A` - яблоко.
- `#` - камень.
- `.` - пустая клетка.
- Матрица хранится как `board[x][y]`, вниз означает увеличение `y`.

## Как думать

Скопируйте карту. Затем повторяйте проходы, пока хотя бы одно яблоко сдвигается. Внутри столбца идите снизу вверх, чтобы нижние яблоки освобождали место, а верхние не проходили сквозь другие яблоки.

## Пример решения

```python
def simulate(board):
    result = []
    for column in board:
        result.append(column[:])

    changed = True
    while changed:
        changed = False
        for x in range(len(result)):
            for y in range(len(result[x]) - 2, -1, -1):
                if result[x][y] == "A" and result[x][y + 1] == ".":
                    result[x][y] = "."
                    result[x][y + 1] = "A"
                    changed = True

    return result
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, в финальном поле под каждым яблоком будет камень, другое яблоко или нижняя граница.

## Частые ошибки

- Сделать только один проход, хотя яблоку надо упасть на несколько клеток.
- Изменить входной `board` и потерять исходные данные.
- Идти сверху вниз и случайно двигать яблоки не так, как задумано.
- Двигать яблоко через камень `#` или другое яблоко.

## Следующий шаг
