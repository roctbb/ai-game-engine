# Лазер и зеркала

## Раздел

**Симуляции мира**. Здесь не нужен BFS, графы или динамическое программирование. Главная цель - в цикле обновлять координаты и направление луча.

## Цель игры

Луч стартует из клетки и движется по направлению `(dx, dy)`. Нужно понять, попадет ли он в цель, отражаясь от зеркал. Стены `#` и поглотители `A` останавливают луч.

## Что видно в визуализации

На поле видны стены, зеркала, поглотитель и цель. Луч рисуется по клеткам, текущая клетка подсвечивается, а внизу показывается направление `(dx, dy)` и тип клетки, на которой находится луч.

## Что написать

Определите `trace(board, start_x, start_y, dx, dy)`. Верните `True`, если луч достиг цели `T`, иначе `False`.

## Входные данные

- `.` - пусто.
- `/` и `\` - зеркала.
- `#` - стена.
- `A` - поглотитель.
- `T` - цель.
- `board[x][y]` - клетка карты.

## Как думать

Это симуляция, а не поиск пути. В цикле смотрите текущую клетку, меняйте направление на зеркале и переходите к следующей клетке. Запоминайте `(x, y, dx, dy)`, чтобы остановить бесконечную петлю: одна и та же клетка с тем же направлением означает, что луч ходит по кругу.

Правила отражения:

```python
if cell == "/":
    dx, dy = -dy, -dx
elif cell == "\\":
    dx, dy = dy, dx
```

## Пример решения

```python
def trace(board, start_x, start_y, dx, dy):
    x = start_x
    y = start_y
    seen = set()

    while 0 <= x < len(board) and 0 <= y < len(board[x]):
        if (x, y, dx, dy) in seen:
            return False
        seen.add((x, y, dx, dy))

        cell = board[x][y]
        if cell == "T":
            return True
        if cell == "#" or cell == "A":
            return False
        if cell == "/":
            dx, dy = -dy, -dx
        elif cell == "\\":
            dx, dy = dy, dx

        x += dx
        y += dy

    return False
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, луч остановится на стене или поглотителе либо попадет в цель.

## Частые ошибки

- Не обработать выход за границы.
- Неправильно отразить направление на `\`.
- Забыть, что `A` останавливает луч.
- Не остановиться при повторе состояния.
- Проверять только `(x, y)`, а направление не учитывать. На зеркалах это может дать ложную петлю.
