# Лазер и зеркала

## Раздел

**Симуляции мира**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

Луч стартует из клетки и движется по направлению `(dx, dy)`. Нужно понять, попадет ли он в цель, отражаясь от зеркал.

## Что видно в визуализации

На поле видны стены, зеркала и цель. Луч рисуется по клеткам, а на зеркалах меняет направление.

## Что написать

Определите `trace(board, start_x, start_y, dx, dy)`. Верните `True`, если луч достиг цели `T`, иначе `False`.

## Входные данные

- `.` - пусто.
- `/` и `\` - зеркала.
- `#` - стена.
- `T` - цель.
- `board[x][y]` - клетка карты.

## Как думать

Это симуляция, а не поиск пути. В цикле смотрите текущую клетку, меняйте направление на зеркале и переходите к следующей клетке. Запоминайте `(x, y, dx, dy)`, чтобы остановить бесконечную петлю.

## Пример решения

```python
def trace(board, start_x, start_y, dx, dy):
    x = start_x
    y = start_y
    seen = set()
    while 0 <= x < len(board) and 0 <= y < len(board[x]):
        if (x, y, dx, dy) in seen:
            return False
        seen.add((x, y, dx, dy))
        cell = board[x][y]
        if cell == "T":
            return True
        if cell == "#":
            return False
        if cell == "/":
            dx, dy = -dy, -dx
        elif cell == "\\":
            dx, dy = dy, dx
        x += dx
        y += dy
    return False
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Не обработать выход за границы.
- Неправильно отразить направление на `\`.
- Не остановиться при повторе состояния.
