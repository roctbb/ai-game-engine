# Арена слабейшего

## Раздел

**Списки и цели**. Здесь не нужен BFS, графы или динамическое программирование. Главная цель - перебрать список и выбрать лучший объект по нескольким признакам.

## Цель игры

На арене несколько врагов. Нужно выбрать лучшую цель:

1. сначала врага с минимальным здоровьем;
2. если здоровье одинаковое - ближайшего;
3. если и расстояние одинаковое - врага с большей наградой.

## Что видно в визуализации

Враги стоят вокруг героя. Над каждым видны HP, расстояние и награда. Выбранная цель подсвечивается, а внизу показано, какие признаки сравнивались.

## Что написать

Определите `choose_target(enemies)`. Верните `id` врага по правилу `hp -> distance -> reward`.

## Входные данные

- `enemies` - список словарей.
- `id` - имя врага.
- `hp` - здоровье. Чем меньше, тем лучше цель.
- `distance` - расстояние до героя. При равном HP ближний враг важнее.
- `reward` - награда. При равных HP и расстоянии выгоднее большая награда.

## Как думать

Возьмите первого врага как лучший вариант. Затем переберите остальных. Заменяйте цель, если нашли меньший HP. Если HP равны, сравните расстояние. Если расстояние тоже равно, сравните награду.

## Пример решения

```python
def choose_target(enemies):
    target = enemies[0]
    for enemy in enemies:
        if enemy["hp"] < target["hp"]:
            target = enemy
        elif enemy["hp"] == target["hp"] and enemy["distance"] < target["distance"]:
            target = enemy
        elif (
            enemy["hp"] == target["hp"]
            and enemy["distance"] == target["distance"]
            and enemy["reward"] > target["reward"]
        ):
            target = enemy
    return target["id"]
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, подсвеченная цель совпадает с ожидаемой, счет доходит до максимума, а финальный кадр показывает успешный выбор.

## Частые ошибки

- Вернуть индекс вместо `id`.
- Искать максимум HP вместо минимума.
- Забыть про расстояние, когда у двух врагов одинаковый HP.
- Выбрать меньшую награду при полном равенстве HP и расстояния.

## Следующий шаг

Попробуйте переписать сравнение через один кортеж: `(hp, distance, -reward)`. Это короткий способ описать тот же порядок выбора.
