# Маг против босса

## Раздел

**Режимы поведения**. Здесь не нужен BFS, графы или динамическое программирование. Главная цель - реагировать на состояние и использовать номер хода.

## Цель игры

Босс действует по повторяющемуся циклу. Маг должен выбрать правильную реакцию на состояние босса и не забывать лечиться каждый 5-й ход, если нет более срочной угрозы.

## Что видно в визуализации

На арене видны маг, босс, шкалы здоровья и текущая фаза атаки. Сильная атака подсвечивается заранее, а сверху идет таймлайн фаз.

## Что написать

Определите `choose_action(turn, boss_state)`. Верните `shield`, `attack`, `area_spell` или `heal`.

## Входные данные

- `turn` - номер хода. Для лечения полезно проверить `turn % 5 == 0`.
- `boss_state` - текущее действие босса, например `heavy_attack` или `summon`.

## Как думать

Сначала реагируйте на опасные состояния: от сильной атаки нужен щит, призыв лучше закрыть массовым заклинанием. После этого проверьте номер хода: каждый 5-й ход маг лечится. В остальные спокойные ходы можно атаковать.

## Пример решения

```python
def choose_action(turn, boss_state):
    if boss_state == "heavy_attack":
        return "shield"
    elif boss_state == "summon":
        return "area_spell"
    elif turn % 5 == 0:
        return "heal"
    return "attack"
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, маг ставит щит против сильной атаки, бьет по призыву массовым заклинанием и лечится на спокойном 5-м ходу.

## Частые ошибки

- Проверять лечение раньше сильной атаки: на опасном ходу важнее щит.
- Игнорировать `turn % 5` и всегда атаковать в спокойный ход.
- Вернуть действие с другим названием.
- Лечиться каждый ход и не наносить урон.
