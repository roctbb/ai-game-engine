# Зельевар

## Раздел

**Счетчики и ресурсы**. Здесь не нужен BFS, графы или динамическое программирование. Главная цель - проверить ограничения по ресурсам и выбрать лучший вариант.

## Цель игры

В лаборатории есть ресурсы и рецепты. Нужно выбрать доступное зелье с максимальной прибылью.

Прибыль считается так:

```python
profit = value - ingredient_cost
```

Стоимость ингредиентов:

- `water` - 1;
- `herb` - 2;
- `crystal` - 5;
- `mushroom` - 3.

## Что видно в визуализации

На столе видны ингредиенты, их стоимость, котлы с рецептами и прибыль каждого зелья. Недоступные рецепты не должны побеждать, даже если у них высокая ценность.

## Что написать

Определите `choose_potion(resources, recipes)`. Верните имя лучшего доступного зелья.

## Входные данные

- `resources` - словарь доступных ингредиентов.
- `recipes` - словарь рецептов.
- В каждом рецепте есть ингредиенты и поле `value` - ценность зелья.

## Как думать

Переберите рецепты. Для каждого рецепта:

1. проверьте, хватает ли всех ингредиентов;
2. посчитайте стоимость ингредиентов;
3. посчитайте прибыль;
4. если зелье доступно и прибыль больше лучшей, запомните его.

## Пример решения

```python
def choose_potion(resources, recipes):
    best_name = None
    best_profit = -1
    costs = {"water": 1, "herb": 2, "crystal": 5, "mushroom": 3}

    for name, recipe in recipes.items():
        can_make = True
        ingredient_cost = 0
        for resource, amount in recipe.items():
            if resource == "value":
                continue
            if resources.get(resource, 0) < amount:
                can_make = False
            ingredient_cost += costs[resource] * amount

        profit = recipe["value"] - ingredient_cost
        if can_make and profit > best_profit:
            best_name = name
            best_profit = profit

    return best_name
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, подсветится доступный рецепт с лучшей прибылью.

## Частые ошибки

- Выбрать самое ценное зелье, но забыть вычесть стоимость ингредиентов.
- Считать `value` ингредиентом.
- Не проверять, хватает ли ресурсов.
- Вернуть весь рецепт вместо имени зелья.
