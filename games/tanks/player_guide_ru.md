# Танки
## Формат карты

Это большая танковая арена из старой версии игры. Карта приходит как список столбцов: первая координата - `x`, вторая - `y`.

```python
width = len(map_state)
height = len(map_state[0]) if width else 0
cell = map_state[x][y]
```

В каждой клетке лежит словарь:

- `player`: танк или башня в клетке, если есть.
- `items`: список предметов в клетке.
- `object`: стена, дерево, вода, база или другой объект, если есть.

В `map_state` также можно читать служебные поля нового протокола:

- `map_state["tick"]`: номер хода.
- `map_state["map"]`: размер карты.
- `map_state["self"]`: позиция вашего танка.
- `map_state["support"]`: состояние поддержки.

## Что нужно написать

В слоте `driver` напишите функцию:

```python
def make_choice(x, y, map_state):
    return "go_right"
```

Функция получает текущие координаты танка и всю карту. Она должна вернуть одно действие:

- `go_up`, `go_down`, `go_left`, `go_right` - поехать на одну клетку.
- `fire_up`, `fire_down`, `fire_left`, `fire_right` - выстрелить в выбранном направлении.
- `use HealthKit`, `use SniperBooster` - использовать предмет из инвентаря, если он есть.
- `up`, `down`, `left`, `right` тоже принимаются и считаются обычным движением для совместимости.

Если несколько танков пытаются проехать в одну клетку, порядок обработки движения случайный: один танк может занять клетку, остальные останутся, если клетка уже занята.

В слоте `support` можно написать:

```python
def make_support(state):
    return "none"
```

Поддержка может вернуть `boost`, чтобы на короткое время увеличить скорость танка, или `none`, чтобы ничего не делать.

## Как читать клетку

```python
def inside(x, y, map_state):
    return 0 <= x < len(map_state) and 0 <= y < len(map_state[x])


def blocked(x, y, map_state):
    if not inside(x, y, map_state):
        return True
    cell = map_state[x][y]
    if cell["player"]:
        return True
    obj = cell["object"]
    return bool(obj and not obj.get("is_transparent", False))
```

Объекты бывают разными: `Wall`, `Tree`, `Water`, `Road`, `Rocks`, `Snow`, `DareAncient`, `RadientAncient`. Предметы: `Coin`, `HealthKit`, `SniperBooster`.

## Первый простой бот

```python
def make_choice(x, y, map_state):
    width = len(map_state)
    height = len(map_state[0]) if width else 0

    # Если справа можно ехать, едем вправо.
    nx = x + 1
    if nx < width:
        cell = map_state[nx][y]
        if not cell["player"] and not cell["object"]:
            return "go_right"

    # Иначе пробуем стрелять вправо.
    return "fire_right"
```

Этот бот слабый, зато показывает главный формат: координаты приходят отдельно, карта читается как `map_state[x][y]`, ответ - строка с действием.

## Как сделать сильнее

1. Сначала научитесь не врезаться в стены, воду, деревья и башни.
2. Потом ищите ближайшие предметы: монеты, аптечки, усилители дальности.
3. Если враг, башня или разрушаемый объект стоит на одной линии без препятствий, стреляйте.
4. Если здоровье низкое, ищите `HealthKit`.
5. Если есть `SniperBooster`, используйте его перед стрельбой по дальним целям.

Для поиска пути по большой карте удобно использовать BFS. В состояние достаточно положить координаты клетки, а соседями считать четыре хода `go_*`.

## Визуализация

Визуализация использует старые PNG-ассеты. Карта большая, поэтому камерой можно управлять:

- стрелки или WASD - двигать камеру;
- колесо мыши, `+`, `-` - менять масштаб;
- `0` или `Home` - сбросить камеру и масштаб.

## Где смотреть примеры

- `examples/coin_shooter.py`: простой бот ищет монеты и стреляет по видимым целям.
- `examples/smart_driver.py`: более аккуратный бот проверяет препятствия и выбирает безопасные ходы.
- `examples/support_boost.py`: пример поддержки с ускорением.

## Ошибки и подсказки

Если функция возвращает неизвестное действие, движок добавит событие `invalid_driver_action`. Для отладки используйте `print(...)`: сообщения появятся в событиях replay рядом с ходами.
