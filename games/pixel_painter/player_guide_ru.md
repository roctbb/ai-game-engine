# Пиксельный художник

## Раздел

**Матрицы и координаты**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

По числовой картинке нужно построить цветную картинку: каждое число заменяется цветом из палитры.

## Что видно в визуализации

Слева показана матрица чисел, справа постепенно появляется пиксельная картинка.

## Что написать

Определите `paint(pixels, palette)`. Верните матрицу цветов в формате `result[x][y]`.

## Входные данные

- `pixels[x][y]` - номер цвета в клетке.
- `palette[number]` - название цвета.
- Первая координата всегда `x`, вторая `y`: карта хранится списком столбцов.

## Как думать

Создайте новую матрицу. Внешний цикл идет по `x`, внутренний по `y`, а в результат кладется `palette[pixels[x][y]]`.

## Пример решения

```python
def paint(pixels, palette):
    result = []
    for x in range(len(pixels)):
        column = []
        for y in range(len(pixels[x])):
            column.append(palette[pixels[x][y]])
        result.append(column)
    return result
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Поменять местами `x` и `y`.
- Изменять входную матрицу без необходимости.
- Забыть создать новый столбец перед внутренним циклом.
