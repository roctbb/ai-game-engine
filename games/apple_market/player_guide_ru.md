# Яблочный рынок

Это соревновательная игра для 2-4 команд. В каждой команде ровно одна роль: `player`.

Движок сам расставляет команды по углам карты. Если команд две, они стартуют в противоположных углах. Если команд четыре, все четыре угла заняты.

## Цель

Собирайте яблоки на карте и относите их на свою базу. На базе яблоки сначала лежат в корзине. Когда в корзине накапливается 4 яблока, они превращаются в 1 литр сока.

Побеждает команда, которая первой накопит на базе 3 литра сока. Если никто не сделал 3 литра за 1200 ходов, выше будет команда с лучшим счетом.

Игрок может нести максимум 3 яблока. Если рюкзак заполнен, обычно выгодно идти к базе.

Важно: яблоки, которые просто лежат на базе, может украсть чужой игрок. Для этого ему надо зайти на вашу базу, пока у него есть свободное место. Готовый сок украсть нельзя.

В центре карты стоит яблоня. Через нее нельзя пройти, зато каждые 5 ходов рядом с ней падают новые яблоки. Яблоня активная, поэтому вокруг центра часто появляется новая добыча, а команды сталкиваются чаще.

## Контракт функции

Напишите функцию:

```python
def make_move(x, y, board, carrying, players, bases):
    ...
```

Параметры:

- `x`, `y` - текущая позиция вашего игрока.
- `board` - карта как список столбцов: первая координата `x`, вторая `y`.
- `carrying` - сколько яблок вы сейчас несете: `0`, `1`, `2` или `3`.
- `players` - список игроков на поле. Этот параметр можно не писать в старых решениях, но с ним легче понять, в кого стоит бросать яблоко.
- `bases` - список баз. В нем видно, сколько яблок можно украсть с базы и сколько безопасного сока уже готово.

Клетку читают так:

```python
cell = board[x][y]
```

`x` растет вправо, `y` растет вниз.

Один элемент списка `players` выглядит так:

```python
{
    "slot": "north_west",
    "name": "Алиса",
    "x": 3,
    "y": 5,
    "carrying": 2,
    "frozen": 0,
    "delivered": 4,
    "is_me": False,
}
```

Самое важное поле для бросков - `carrying`. Если соперник несет `0` яблок, замораживать его обычно менее выгодно. Если несет `1`, `2` или `3`, бросок может помешать ему донести яблоки на базу.

Один элемент списка `bases` выглядит так:

```python
{
    "slot": "north_west",
    "name": "Алиса",
    "x": 1,
    "y": 1,
    "apples": 3,
    "juice": 1,
    "is_me": False,
}
```

- `apples` - яблоки в корзине базы. Их можно украсть, если это чужая база.
- `juice` - готовый сок. Его украсть нельзя.
- `is_me` - `True` у вашей базы.

## Значения на карте

- `-1` - стена.
- `-2` - любой соперник.
- `0` - пустая клетка.
- `1` - яблоко.
- `2` - ваша база.
- `3` - яблоня в центре карты. Это препятствие, но рядом с ним появляются новые яблоки.

Каждая команда видит свою базу как `2`. Координаты всех баз удобнее брать из параметра `bases`.

## Действия

Функция должна вернуть одну строку:

- `"up"` - шаг вверх.
- `"down"` - шаг вниз.
- `"left"` - шаг влево.
- `"right"` - шаг вправо.
- `"stay"` - стоять на месте.
- `"throw_up"` - бросить яблоко вверх.
- `"throw_down"` - бросить яблоко вниз.
- `"throw_left"` - бросить яблоко влево.
- `"throw_right"` - бросить яблоко вправо.

Бросок работает только если `carrying > 0`. Один бросок всегда тратит одно яблоко: и при попадании, и при промахе. Если вернуть `"throw_right"` или другое действие броска, когда `carrying == 0`, броска не будет, а ход засчитается как ошибка.

Яблоко летит по прямой до 5 клеток, останавливается об стену и замораживает первого соперника на пути на 3 следующих хода.

Замороженный игрок пропускает ходы, поэтому бросок полезен, когда соперник стоит на одной линии с вами.

В реплее бросок показывается отдельно: видно имя игрока, который кинул яблоко, траекторию полета, потерю одного яблока и подпись `ПОПАДАНИЕ: freeze`, если соперник был заморожен.

Если несколько игроков одновременно идут в одну свободную клетку, движок случайно выбирает одного игрока, который займет ее. Остальные остаются на местах. Обмен местами блокируется.

## Первый план решения

1. Если рядом по прямой есть соперник, он несет яблоки, и вы тоже несете яблоко, бросьте в него.
2. Если `carrying == 3`, найдите путь к своей базе `2`.
3. Если на чужой базе лежат яблоки и у вас есть свободное место, можно попробовать украсть их.
4. Иначе найдите путь к ближайшему яблоку `1`.
5. Если яблок сейчас не видно, можно двигаться ближе к центру или к базе: скоро рядом с яблоней появятся новые яблоки.
6. Если цели нет, сделайте любой безопасный ход или верните `"stay"`.

Карта и яблоки генерируются случайно, поэтому нельзя заранее записать готовую последовательность ходов.

## Простой пример

Этот пример не ищет лучший путь. Он показывает формат функции и не является готовым решением для победы.

```python
DIRECTIONS = {
    "up": (0, -1),
    "right": (1, 0),
    "down": (0, 1),
    "left": (-1, 0),
}


def in_bounds(x, y, board):
    return 0 <= x < len(board) and 0 <= y < len(board[x])


def can_step(x, y, board):
    return in_bounds(x, y, board) and board[x][y] not in (-2, -1, 3)


def opponent_carrying_at(x, y, players):
    if not players:
        return 1

    for player in players:
        if player["is_me"]:
            continue
        if player["x"] == x and player["y"] == y:
            return player["carrying"]

    return 0


def try_throw(x, y, board, carrying, players):
    if carrying <= 0:
        return None

    for action, (dx, dy) in DIRECTIONS.items():
        tx = x
        ty = y
        for _ in range(5):
            tx += dx
            ty += dy
            if not in_bounds(tx, ty, board) or board[tx][ty] in (-1, 3):
                break
            if board[tx][ty] == -2 and opponent_carrying_at(tx, ty, players) > 0:
                return "throw_" + action
    return None


def make_move(x, y, board, carrying, players, bases):
    throw_action = try_throw(x, y, board, carrying, players)
    if throw_action:
        return throw_action

    for action, (dx, dy) in DIRECTIONS.items():
        nx = x + dx
        ny = y + dy
        if can_step(nx, ny, board):
            return action

    return "stay"
```

## Идея сильнее: BFS

Для хорошей стратегии удобно искать кратчайший путь:

- если несете 3 яблока, цель - база `2`;
- если у чужой базы есть `apples > 0`, можно идти туда и воровать;
- иначе цель - ближайшее яблоко `1`;
- соперников `-2` лучше считать занятыми клетками;
- стены `-1` и яблоня `3` непроходимы.

Пример с BFS есть в папке `examples`.

## Где смотреть примеры

- `examples/demo.py` - базовая стратегия для одной роли `player`.
- `examples/north_bfs.py` - BFS с бросками.
- `examples/south_bfs.py` - BFS с другим порядком обхода и бросками.

Даже если файл называется `north_bfs.py` или `south_bfs.py`, в новой версии это просто разные демо-стратегии для одного слота `player`.

## Как начисляются очки

Главное - готовый сок. 1 литр сока дает большой бонус, яблоки в корзине базы дают меньший бонус, потому что их еще можно украсть. Также есть небольшой бонус за удачные броски и штраф за неправильные ходы.

Полезные данные в результате матча:

- `scores` - счет команд.
- `winner_slots` - победившие внутренние позиции.
- `placements` - места команд.
- `metrics.delivered` - сколько яблок игроки принесли на базу всего.
- `metrics.base_apples` - сколько яблок сейчас лежит на каждой базе и еще может быть украдено.
- `metrics.juice` - сколько литров сока готово на каждой базе.
- `metrics.apples_per_juice` - сколько яблок нужно на 1 литр сока.
- `metrics.juice_to_win` - сколько литров нужно для победы.
- `metrics.turn_limit` - лимит ходов матча.
- `metrics.apples_spawned_total` - сколько яблок всего появилось за матч.
- `metrics.throws` - сколько бросков сделал каждый игрок.
- `metrics.frozen` - сколько ходов заморозки осталось к концу матча.
- `metrics.invalid_moves` - сколько неправильных действий сделал игрок, включая попытку бросить яблоко с пустыми руками.

## Частые ошибки

- Перепутать координаты и написать `board[y][x]` вместо `board[x][y]`.
- Идти в стену `-1`.
- Идти в яблоню `3`: это тоже препятствие.
- Бросать яблоко, когда `carrying == 0`.
- Искать только одного конкретного соперника: на карте любой соперник отмечен как `-2`.
- Забыть вернуться на базу, когда уже несете 3 яблока.
- Не учитывать `bases`: чужая база с яблоками может быть выгодной целью.
- Думать, что сок можно украсть. Украсть можно только яблоки, которые еще не превратились в сок.
