# Патрульный страж

## Раздел

**Режимы поведения**. Задача показывает, как программа может помнить состояние между шагами.

## Цель игры

Страж идет по маршруту. Он атакует врага, возвращается на маршрут, если сбился, и расследует шум, когда тревога накопилась.

## Что видно в визуализации

На поле показан маршрут патруля, события на нем и текущий режим стража. При шуме растет шкала тревоги, при враге включается атака.

## Входные данные

`events` - список событий. Каждое событие - словарь:

```python
{"type": "noise", "noise": 2}
```

Типы:

- `clear` - все спокойно.
- `noise` - шум, добавляет тревогу.
- `enemy` - враг.
- `lost_route` - страж сбился с маршрута.

## Как думать

Заведите переменную `alert = 0`. Если событие `noise`, прибавьте его силу к тревоге. Если тревога стала 3 или больше, верните `investigate`. Враг и потеря маршрута важнее шума.

## Пример решения

```python
def choose_actions(events):
    actions = []
    alert = 0

    for event in events:
        if event["type"] == "enemy":
            actions.append("attack")
            alert = 0
        elif event["type"] == "lost_route":
            actions.append("return_to_route")
            alert = 0
        elif event["type"] == "noise":
            alert += event["noise"]
            if alert >= 3:
                actions.append("investigate")
            else:
                actions.append("patrol")
        else:
            actions.append("patrol")

    return actions
```

## Частые ошибки

- Не хранить тревогу между событиями.
- Не сбрасывать тревогу после врага или возврата на маршрут.
- Вернуть одну строку вместо списка действий.
