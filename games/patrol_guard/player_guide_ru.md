# Патрульный страж

## Раздел

**Режимы поведения**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

Страж получает события по порядку и выбирает действие: патрулировать, атаковать или вернуться на маршрут.

## Что видно в визуализации

На карте показан маршрут патруля, тревожные события и режим стража. При враге страж выходит из маршрута, потом возвращается.

## Что написать

Определите `choose_actions(events)`. Верните список действий `patrol`, `attack` или `return_to_route`.

## Входные данные

- `events` - список событий.
- В событии может быть враг, состояние маршрута или необходимость вернуться.
- На каждое событие нужен один ответ.

## Как думать

Думайте режимами. Если враг рядом - атака, если страж сбился с маршрута - возврат, иначе патруль.

## Пример решения

```python
def choose_actions(events):
    actions = []
    for event in events:
        if event == "enemy":
            actions.append("attack")
        elif event == "lost_route":
            actions.append("return_to_route")
        else:
            actions.append("patrol")
    return actions
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Вернуть одно действие вместо списка.
- Не сохранять порядок событий.
- Всегда атаковать, даже когда врага нет.
