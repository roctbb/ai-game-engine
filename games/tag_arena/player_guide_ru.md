# Охотник и беглец

Это командная соревновательная игра. У каждой команды два слота:

- `runner` — беглец;
- `hunter` — охотник.

Это сделано для честности: каждая команда одновременно отвечает и за сбор звезд, и за поимку чужого беглеца.

## Контракт функции

В каждом слоте напишите:

```python
def make_move(x, y, board):
    ...
```

Функция возвращает одно действие:

```python
"up"
"down"
"left"
"right"
"stay"
```

## Координаты

Карта передается как список столбцов:

```python
cell = board[x][y]
```

`x` растет вправо, `y` растет вниз.

## Значения в `board`

```python
-1  # стена
-2  # важный соперник
 0  # пустая клетка
 1  # звезда
```

Для `runner` клетка `-2` — чужой охотник. Лучше туда не ходить.

Для `hunter` клетка `-2` — чужой беглец. Его нужно догнать.

## Правила

- Беглец собирает звезды для своей команды.
- Охотник ловит чужого беглеца.
- Если охотник оказался на клетке чужого беглеца, команда охотника получает очки за поимку.
- Пойманный беглец возвращается на старт и пропускает несколько ходов.
- Ходы всех ролей считаются одновременно.
- Если персонажи разных команд хотят занять одну и ту же свободную клетку, движок случайно выбирает одного, остальные остаются на месте.
- Если персонажи пытаются поменяться местами, движение блокируется.

## Простые стратегии

Беглец:

1. Найти ближайшую звезду `1`.
2. Построить путь через BFS.
3. Не ходить в стены `-1` и в клетку чужого охотника `-2`.

Охотник:

1. Найти чужого беглеца `-2`.
2. Построить путь через BFS.
3. Сделать первый шаг по пути.

## Примеры

- `examples/runner_bfs.py` — беглец идет к ближайшей звезде.
- `examples/hunter_bfs.py` — охотник идет к чужому беглецу.

В визуализации каждый персонаж подписан: команда и роль. `Б` означает беглец, `О` означает охотник.
