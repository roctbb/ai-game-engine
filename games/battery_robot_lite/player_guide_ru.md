# Батарейный робот Lite

## Раздел

**Счетчики и ресурсы**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

Робот принимает одно решение по состоянию: почистить грязь, зарядиться, ехать дальше или ждать.

## Что видно в визуализации

На поле виден робот, грязная клетка, зарядная станция и батарейка прямо рядом с персонажем.

## Что написать

Определите `choose_action(state)`. Верните `clean`, `charge`, `move` или `wait`.

## Входные данные

- `state["battery"]` - заряд.
- `state["dirty"]` - есть ли грязь на клетке.
- `state["on_charger"]` - стоит ли робот на зарядке.
- `state["front_clear"]` - можно ли ехать вперед.

## Как думать

Сначала проверяйте критичный ресурс. Если заряд низкий и робот на зарядке, лучше зарядиться. Чистка стоит дороже движения.

## Пример решения

```python
def choose_action(state):
    if state["battery"] <= 2 and state["on_charger"]:
        return "charge"
    elif state["dirty"] and state["battery"] >= 2:
        return "clean"
    elif state["front_clear"] and state["battery"] >= 1:
        return "move"
    return "wait"
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Пытаться чистить без батареи.
- Ехать вперед, когда проход закрыт.
- Не вернуть `wait` для безвыходной ситуации.
