# Рюкзак шахтера

## Раздел

**Счетчики и ресурсы**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

Шахтер идет по жиле и кладет руду в рюкзак. Нужно понять, сколько кусков подряд можно взять, пока рюкзак не перегружен.

## Что видно в визуализации

Видно тоннель, куски руды разного веса и полоску заполнения рюкзака у шахтера.

## Что написать

Определите `solve(ores, capacity)`. Верните число взятых кусков руды.

## Входные данные

- `ores` - список весов руды по порядку.
- `capacity` - максимальный вес рюкзака.

## Как думать

Храните два счетчика: общий вес и количество взятых кусков. Если следующий кусок не помещается, остановите цикл.

## Пример решения

```python
def solve(ores, capacity):
    total = 0
    count = 0
    for ore in ores:
        if total + ore > capacity:
            break
        total += ore
        count += 1
    return count
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Считать все куски, даже после перегруза.
- Проверять перегруз после добавления и не отменять последний кусок.
- Вернуть вес вместо количества.
