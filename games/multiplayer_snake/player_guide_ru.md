# Змейка
## Общий формат карты

Во всех задачах с полем карта передается как список столбцов: первая координата - `x`, вторая - `y`.

```python
width = len(board)
height = len(board[0]) if width else 0
cell = board[x][y]
```

`x` растет вправо, `y` растет вниз. Перед проверкой клетки всегда проверяйте границы: `0 <= x < width` и `0 <= y < height`.

В этой игре четыре игрока управляют каждый своей змейкой. Каждый игрок пишет функцию `make_move(x, y, board)`.

- `x`, `y` — координаты головы вашей змейки.
- `board` — общая матрица поля.

Значения в `board`:

- `0` — пустая клетка.
- `1` — еда. На поле может быть несколько яблок одновременно.
- `-1` — стена или средняя часть тела любой змейки.
- `-2` — голова любой живой змейки, включая вашу.
- `-3` — хвост любой живой змейки.

Функция возвращает `up`, `down`, `left` или `right`.

```python
def make_move(x, y, board):
    for action, dx, dy in [("right", 1, 0), ("down", 0, 1), ("left", -1, 0), ("up", 0, -1)]:
        nx = x + dx
        ny = y + dy
        if 0 <= nx < len(board) and 0 <= ny < len(board[nx]) and board[nx][ny] >= 0:
            return action
    return "right"
```

Все четыре змейки ходят одновременно.

Правила столкновений:

- Ход в край поля или тело любой змейки убивает змейку, но матч продолжается.
- Если несколько змеек идут в одну клетку, все участники столкновения погибают.
- Если две головы меняются местами, обе змейки погибают.
- Разворот на 180 градусов блокируется: вместо него остается прежнее направление.
- В освобождающийся хвост своей змейки заходить можно, если змейка в этот ход не ест.
- Клетка `-3` показывает хвост, но он освобождается не всегда: если змейка ест, хвост остается на месте.
- При смерти из тела выпадают яблоки, а змейка через короткую паузу возрождается в свободном месте, чаще всего из норки в своем углу.
- Набранные очки, съеденные яблоки и штрафы после смерти не сбрасываются.

Итоговый счет: еда важнее всего, затем текущая и максимальная длина. Смерти и неправильные ходы дают штраф.

## Подробный разбор

### Что нужно написать

Это соревновательный матч на четыре игрока: все запускаются на одной карте, каждый управляет одной змейкой. Результат сравнивается по очкам.

Главный контракт: `Определите функцию make_move(x, y, board), возвращающую up/down/left/right.`

Перед тем как писать сильную стратегию, убедитесь, что функция всегда возвращает действие правильного типа. Даже слабый, но стабильный бот лучше, чем бот, который иногда падает с ошибкой.

### Как читать карту и координаты

В задачах с картой используйте порядок координат `board[x][y]`: сначала столбец `x`, потом строка `y`. Это отличается от привычной записи в тетради, поэтому полезно сразу завести маленькие функции `in_bounds` и `is_safe`.

```python
DIRECTIONS = {
    "up": (0, -1),
    "right": (1, 0),
    "down": (0, 1),
    "left": (-1, 0),
}


def in_bounds(x, y, board):
    return 0 <= x < len(board) and 0 <= y < len(board[x])


def is_safe(x, y, board):
    return in_bounds(x, y, board) and board[x][y] >= 0
```


### Первый рабочий алгоритм

1. Сохраните все разрешенные действия в список.
2. Для каждого действия посчитайте новую позицию.
3. Отбросьте действия, которые ведут в опасные клетки: любое значение меньше `0`.
4. Из оставшихся выберите действие, которое приближает к еде.
5. Если еды не видно, выберите любой безопасный ход.

### Как сделать решение сильнее

В матче все игроки ходят одновременно. Полезно думать в два шага: что я получу этим ходом и куда могут прийти соперники после моего хода.

Полезный прием для отладки: печатайте короткие сообщения, например выбранную цель и итоговое действие. Тогда по replay легче понять, почему бот сделал именно этот ход.

### На что обратить внимание

Не выбирайте ход, после которого у змейки не останется свободных соседних клеток. Клетки рядом с `-2` опасны: туда может пойти чужая голова на этом же ходу.

Для первого улучшения попробуйте сравнить свою стратегию с демо-ботом и посмотреть, в каких ситуациях она теряет очки.

### Где смотреть примеры

- `examples/food_chaser.py`: Демо: охотник за едой. Идет к еде и избегает опасных клеток.

## Быстрый старт и простой пример

Это соревновательный матч на четыре змейки. Начните с простого решения, убедитесь, что оно запускается, а потом постепенно улучшайте стратегию.

### Контракт функции

Определите функцию make_move(x, y, board), возвращающую up/down/left/right. Каждый игрок управляет одной змейкой в матче на четыре участника.

Координаты: `x` растет вправо, `y` растет вниз. Карта хранится как список столбцов: `board[x][y]`.

### Минимальный пример

```python
DIRECTIONS = {
    "up": (0, -1),
    "right": (1, 0),
    "down": (0, 1),
    "left": (-1, 0),
}


def make_move(x, y, board):
    for action, (dx, dy) in DIRECTIONS.items():
        nx = x + dx
        ny = y + dy
        if 0 <= nx < len(board) and 0 <= ny < len(board[nx]):
            cell = board[nx][ny]
            if cell >= 0:
                return action
    return "up"
```

### Как улучшать решение

1. Сначала выпишите, какие данные приходят в функцию: координаты и карта.
2. Сделайте проверку допустимых ответов: бот не должен ходить в стену.
3. Заведите функции проверки границ и чтения клетки через `board[x][y]`.
4. Сравните свое решение с примером из `examples`, запустите несколько раз и посмотрите replay.

### Где посмотреть полный пример

- `examples/food_chaser.py` для слота `player`: охотник за едой.


## Баланс матча

Ходы всех игроков считаются одновременно. Игра заканчивается по лимиту ходов или когда суммарная площадь живых змеек становится достаточно большой. Если несколько игроков набрали одинаковый счет, возможна ничья: смотрите `winner_slots`, `is_tie`, `scores` и `placements`.


## Ошибки и подсказки

Если бот возвращает неправильную команду или идет в стену, движок добавляет событие с полем `message`. Это короткое объяснение можно показывать участнику в интерфейсе.
