# Лучник на стене

## Раздел

**Условия и выбор**. Эта задача стоит в начале курса: здесь не нужен BFS, графы или динамическое программирование. Главная цель - потренировать базовый прием Python на понятной игровой сцене.

## Цель игры

Лучник защищает ворота. Враги бегут к стене, а игрок решает: стрелять, перезаряжаться, поднять щит или ждать.

## Что видно в визуализации

На стене стоит лучник, к воротам движется враг, стрелы летят по дорожке. Рядом показаны стрелы и здоровье.

## Что написать

Определите `choose_action(enemy, arrows, hp)`. Верните `shoot`, `reload`, `shield` или `wait`.

## Входные данные

- `enemy["distance"]` - расстояние до врага.
- `enemy["is_aiming"]` - готовится ли враг стрелять.
- `arrows` - сколько стрел осталось.
- `hp` - здоровье лучника.

## Как думать

Сначала проверьте ресурс: без стрел нельзя стрелять. Затем оцените опасность: близкого врага лучше атаковать, при угрозе и малом HP полезен щит.

## Пример решения

```python
def choose_action(enemy, arrows, hp):
    if arrows <= 0:
        return "reload"
    elif enemy["distance"] <= 3:
        return "shoot"
    elif enemy["is_aiming"] and hp < 50:
        return "shield"
    return "wait"
```

## Как проверить себя

Запустите пример и посмотрите на анимацию. Если решение правильное, действия на поле совпадают с ожидаемыми шагами, счет доходит до максимума, а финальный кадр показывает победу или успешный выбор.

## Частые ошибки

- Стрелять при `arrows == 0`.
- Игнорировать близкого врага.
- Возвращать `attack` вместо разрешенного `shoot`.
