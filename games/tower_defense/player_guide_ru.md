# Защита базы башнями
## Общий формат карты

Во всех задачах с полем карта передается как список столбцов: первая координата - `x`, вторая - `y`.

```python
width = len(board)
height = len(board[0]) if width else 0
cell = board[x][y]
```

`x` растет вправо, `y` растет вниз. Перед проверкой клетки всегда проверяйте границы: `0 <= x < width` и `0 <= y < height`.

Напишите функцию `place_tower(state)`, которая возвращает номер линии для постройки башни: `0`, `1` или `2`.

В `state` приходят:

- `tick`: номер текущего хода.
- `lanes`: число линий.
- `track_length`: длина пути врагов.
- `tower_cost`: цена одной башни.
- `base_hp`: здоровье базы.
- `budget`: доступный бюджет.
- `towers`: сколько башен стоит на каждой линии.
- `enemies`: список врагов с линией, позицией и здоровьем.

Цель: сохранить базу и уничтожить как можно больше врагов. Партия длится до 44 ходов: первые волны простые, затем враги идут плотнее, иногда по два за ход, а их здоровье постепенно растет. Башня стоит `state["tower_cost"]` единиц бюджета; если бюджета не хватает, действие не строит башню.

Для отладки можно использовать `print(...)`. Вывод появится в консоли под визуализацией синхронно с кадрами replay.

## Подробный разбор

### Что нужно написать

Это одиночная задача: один ваш алгоритм управляет попыткой от начала до конца.

Главный контракт: `Определите функцию place_tower(state), возвращающую lane 0..2 для постройки башни, и защитите базу.`

Перед тем как писать сильную стратегию, убедитесь, что функция всегда возвращает действие правильного типа. Даже слабый, но стабильный бот лучше, чем бот, который иногда падает с ошибкой.

### Как читать карту и координаты

Здесь нет обычной матрицы `board`: вся информация приходит в `state`. Сначала достаньте `budget`, `lanes`, `towers` и `enemies`, затем верните номер линии для новой башни или `None`, если строить нельзя.

### Первый рабочий алгоритм

1. Если бюджета меньше `state["tower_cost"]`, верните `None`.
2. Для каждой линии посчитайте давление: сколько врагов на линии, как близко они к базе и сколько у них здоровья.
3. Добавьте небольшой штраф линиям, где уже слишком много башен.
4. Выберите линию с самым большим давлением.
5. После нескольких запусков сравните, не проигрывает ли база из-за слабой линии в поздних волнах.

Для этой игры хорошая основная идея: жадная оценка линий: стройте башню там, где враги ближе к базе или где урон сейчас важнее всего.

### Как сделать решение сильнее

В одиночной задаче удобно улучшать решение постепенно: сначала безопасный бот, затем бот с поиском пути, затем бот, который учитывает очки, ресурсы и будущую опасность.

Полезный прием для отладки: печатайте короткие сообщения, например выбранную цель, оценку вариантов и итоговое действие. Тогда по replay легче понять, почему бот сделал именно этот ход.

### На что обратить внимание

Не стройте все башни в одну линию, если другая линия уже близка к поражению. Простая стратегия "по очереди 0, 1, 2" теперь может пережить часть волн, но обычно теряет здоровье базы на плотных атаках. Лучше смотреть на реальных врагов в `state["enemies"]`.

Для первого самостоятельного улучшения попробуйте добиться, чтобы бот всегда возвращал разрешенную команду и не падал с ошибкой. После каждого изменения запускайте пример несколько раз: стратегия не должна зависеть от одного заранее угаданного хода.

### Где смотреть примеры

- `examples/defender_balanced.py`: Демо: баланс по линиям. Ставит башню на линию с максимальным давлением врагов.

## Быстрый старт и простой пример

Эта игра - одиночная задача. Начните с простого решения, убедитесь, что оно запускается, а потом постепенно улучшайте стратегию.

### Контракт функции

Определите функцию place_tower(state), возвращающую lane 0..2 для постройки башни, и защитите базу.

Координаты в играх с сеткой читаются одинаково: `x` растет вправо, `y` растет вниз. Если поле называется `board` или `maze`, карта хранится как список столбцов: `len(board)` — ширина, `len(board[0])` — высота, клетку читают как `board[x][y]` или `maze[x][y]`. Если в игре используется `state`, внутри него обычно есть текущая позиция, карта и цель.

### Минимальный пример

Этот пример специально простой. Он показывает формат функции и ответа, но не пытается быть сильной стратегией.

```python
def place_tower(state):
    tower_cost = state.get("tower_cost", 6)
    if state["budget"] < tower_cost:
        return None

    # Строим башню на линии, где сейчас больше всего врагов.
    counts = [0, 0, 0]
    for enemy in state["enemies"]:
        counts[enemy["lane"]] += 1
    return counts.index(max(counts))
```

### Как улучшать решение

1. Сначала выпишите, какие данные реально приходят в функцию: координаты, карта, ресурс, счет, роль или `state`.
2. Сделайте проверку допустимых ответов: бот не должен ходить в стену, открывать уже открытую клетку или строить башню без бюджета.
3. Для задач с картой заведите маленькие функции проверки границ и чтения клетки через `board[x][y]`.
4. Если есть ключи, время, вода, огонь, батарейка, ящики или соперник, храните это в состоянии алгоритма, а не только координаты.
5. Сравните свое решение с примером из `examples`, запустите несколько случайных карт и посмотрите replay.

### Где посмотреть полный пример

- `examples/defender_balanced.py` для слота `defender`: Демо: баланс по линиям.

Полный пример можно открыть в папке `examples` рядом с этой документацией. Его удобно запускать как рабочую основу, а затем упрощать или усиливать под свою стратегию.


## Ошибки и подсказки

Если бот возвращает неправильную команду, идет в стену или ему не хватает ресурса, движок добавляет событие с полем `message`. Это короткое объяснение можно показывать участнику в интерфейсе.
