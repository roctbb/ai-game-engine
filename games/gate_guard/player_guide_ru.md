# Страж ворот

## Раздел

**Условия и выбор**. Здесь тренируются `if`, `elif`, `else` и порядок проверок.

## Цель игры

Герой стоит перед воротами. В одной ситуации может быть сразу несколько признаков: рядом враг, есть ловушка, ворота закрыты, ключ есть или его нет. Нужно вернуть одно действие.

## Что видно в визуализации

На поле показаны герой, ворота, ловушка, ключ и расстояние до врага. Если действие удачное, подсветка совпадает с ожидаемым выбором.

## Что написать

Определите функцию:

```python
def choose_action(gate):
    ...
```

Функция должна вернуть строку: `attack`, `disarm`, `use_key`, `open` или `wait`.

## Входные данные

- `gate["enemy_distance"]` - расстояние до врага.
- `gate["trap"]` - есть ли ловушка.
- `gate["trap_damage"]` - урон ловушки.
- `gate["hp"]` - здоровье героя.
- `gate["locked"]` - закрыты ли ворота.
- `gate["has_key"]` - есть ли ключ.

## Правило выбора

Проверяйте признаки по приоритету:

1. Если враг рядом (`enemy_distance <= 1`) - `attack`.
2. Иначе если есть ловушка и герой переживет ее урон - `disarm`.
3. Иначе если ворота закрыты и есть ключ - `use_key`.
4. Иначе если ворота закрыты, но ключа нет - `wait`.
5. Иначе - `open`.

## Как думать

Сначала задайте вопрос: "Что опаснее всего прямо сейчас?" Враг рядом важнее замка, ловушка важнее обычного открытия, а закрытые ворота без ключа лучше не трогать.

## Пример решения

```python
def choose_action(gate):
    if gate["enemy_distance"] <= 1:
        return "attack"
    elif gate["trap"] and gate["trap_damage"] < gate["hp"]:
        return "disarm"
    elif gate["locked"] and gate["has_key"]:
        return "use_key"
    elif gate["locked"]:
        return "wait"
    return "open"
```

## Частые ошибки

- Сначала открывать ворота, а уже потом проверять врага.
- Обезвреживать слишком опасную ловушку.
- Забыть случай закрытых ворот без ключа.
